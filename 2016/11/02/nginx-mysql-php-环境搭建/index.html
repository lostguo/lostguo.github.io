nginx+mysql+php 环境搭建

一直安装lnmp环境乱糟糟的,这次以找到的一个还不错的方法并结合自己的研究配置lnmp,特此记录。

参考地址 编译安装nginx1.9.7+php7.0.0服务器环境

我的系统

shell> cat /etc/redhat-release
CentOS Linux release 7.2.1511 (Core)


nginx的安装

依赖

lnmp环境搭建依赖安装必不可少,不然就是各种麻烦,所以一口气安装好吧0.0

shell> yum -y install gcc gcc-c++ autoconf automake libtool make cmake
shell> yum -y install zlib zlib-devel openssl openssl-devel pcre-devel

zlib: 为nginx提供gzip模块，需要zlib库支持

创建nginx用户组

shell> groupadd nginx
shell> useradd -g nginx -M nginx

-g为用户指定组 -M不自动生成home目录,毕竟我们不是为了创建一个用户

shell> vi /etc/passwd

找到nginx，将后面的/bin/bash改为/sbin/nologin就不让ssh登录啦。

编译安装nginx

进入nginx官网啦,找到最新版本的tar.gz包的链接,执行以下动作。

shell> wget http://xxxxxxxxxx/nginx1.7.x.tar.gz
shell> tar zxvf nginx1.7.x.tar.gz
shell> cd nginx1.7.x

执行./configure,把软件安装到/usr/local目录,用conf、etc、sbin、run等进行归类管理。

shell> ./configure --prefix=/usr/local/nginx \
--pid-path=/usr/local/nginx/run/nginx.pid \
--with-http_ssl_module \
--user=nginx \
--group=nginx \
--with-pcre \
--without-mail_pop3_module \
--without-mail_imap_module \
--without-mail_smtp_module

最后三项为禁用nginx作为邮件代理服务器,建议想搭建邮件服务器的同志去阅读nginx搭建邮件服务器的教程。
检查./configure结果,看看有没有报错,有没有缺少模块,如果一切OK，那么往下继续，如果感觉不对，可以用./configure –help认真阅读一下。

shell> make
shell> make install

表急,安装完了,但是nginx还没有运行

shell> cd /usr/local/nginx
shell> ls
shell> sbin/nginx


php7的编译安装

依赖环境

shell> yum -y install libxml2 libxml2-devel openssl openssl-devel curl-devel libjpeg-devel libpng-devel freetype-devel libmcrypt-devel

编译安装php7

shell> wget http://am1.php.net/get/php-7.0.0.tar.gz/from/this/mirror
shell> tar zvxf php-7.0.0.tar.gz
shell> cd php-7.0.0

接下来要进行编译前的配置,我们需要提供php-fpm相关参数

shell> ./configure --prefix=/usr/local/php7 \
--with-config-file-path=/usr/local/php7/etc \
--with-config-file-scan-dir=/usr/local/php7/etc/php.d \
--with-mcrypt=/usr/include \
--enable-mysqlnd \
--with-mysqli \
--with-pdo-mysql \
--enable-fpm \
--with-fpm-user=nginx \
--with-fpm-group=nginx \
--with-gd \
--with-iconv \
--with-zlib \
--enable-xml \
--enable-shmop \
--enable-sysvsem \
--enable-inline-optimization \
--enable-mbregex \
--enable-mbstring \
--enable-ftp \
--enable-gd-native-ttf \
--with-openssl \
--enable-pcntl \
--enable-sockets \
--with-xmlrpc \
--enable-zip \
--enable-soap \
--without-pear \
--with-gettext \
--enable-session \
--with-curl \
--with-jpeg-dir \
--with-freetype-dir \
--enable-opcache

配置无误后执行:

shell> make
shell> make install

如果安装成功，那么php7的安装就OK了。

调整php配置

安装好后发现/usr/local/php7/etc下面没有php.ini文件,去php7的源码安装包里找吧,那里有着php7全部的宝藏

shell> cd /usr/src/php-7.0.0/
shell> ls

惯例你会发现有俩个php.ini-xxx的文件,一个是产品模式,一个是开发模式

shell> cp php.ini-production /usr/local/php7/etc/php.ini
shell> vi /usr/local/php7/etc/php.ini

vi进行自己所需的其他配置。

启用php-fpm服务

上面我们在编译php7的时候，已经将fpm模块编译了，那么接下来，我们要启用php-fpm。但是默认情况下它的配置文件和服务都没有启用，所以要我们自己来搞定。

搞定配置文件

shell> cd /usr/local/php7/etc
shell> mv php-fpm.conf.default php-fpm.conf
shell> mv php-fpm.d/www.conf.default php-fpm.d/www.conf

php-fpm 的开启我直接通过systemctl start php-fpm 开启了0.0。
可以通过ps -ef | grep php-fpm来查看php-fpm的运行状态。

nginx和php-fpm配置

php-fpm走的是9000端口,外网是无法访问的,我们需要在nginx的配置文件中增加代理的规则，即可让用户在访问80端口，请求php的时候，交由后端的fpm去执行。

shell> vi /usr/local/nginx/conf/nginx.conf

应该在nginx.conf已经有例子了,如下:

#location ~ \.php$ {
  #   root           html;
  #  fastcgi_pass   127.0.0.1:9000;
  #  fastcgi_index  index.php;
  #  fastcgi_param  SCRIPT_FILENAME  /script$fastcgi_script_name;
  #  include        fastcgi_params;
#}

把前面的#注释符号去掉，把script改为$document_root

location ~ \.php$ {
    root           html;
    fastcgi_pass   127.0.0.1:9000;
    fastcgi_index  index.php;
    fastcgi_param  SCRIPT_FILENAME  /$document_root$fastcgi_script_name;
    include        fastcgi_params;
}

这样就OK了，重新载入nginx配置。

shell> /usr/local/nginx/sbin/nginx -s reload

到/usr/local/nginx/html去写个文件测试一下吧。

Mysql的安装

在下安装了mysql5.7 官网描述的挺详细。
各位看官请看准自己系统官网安装说明传送门

官方源

请进入官网寻找自己版本所需的源,官网源传送门
由于在下是centos7.2,内核是Red Hat Enterprise Linux 7,所以选择了mysql57-community-release-el7-9.noarch.rpm,以下以此示例

下载源

shell> wget http://dev.mysql.com/get/mysql57-community-release-el7-9.noarch.rpm

安装源

shell> sudo yum localinstall mysql57-community-release-el7-9.noarch.rpm

检查

shell> yum repolist enabled | grep "mysql.*-community.*"

安装

shell> sudo yum install mysql-community-server

启动

shell> sudo service mysqld start

抑或

shell> systemctl start mysqld

检查

shell> sudo service mysqld status

要显示running ,类似如下

Active: active (running) since Fri 2016-10-28 10:28:42 CST; 5 days ago

如果正常运行,那么恭喜你,你已经成功的安装完毕了mysql

初始密码

系统创建了一个超级账户’root’@’localhost的,和以前不太一样的是,mysql -uroot -p 不能空密码进,它的初始密码存在了错误日志里。

shell> sudo grep 'temporary password' /var/log/mysqld.log

即可查看到初始密码

2016-10-28T02:28:38.449839Z 1 [Note] A temporary password is generated for root@localhost: 8Ba=2gtz+?wx

类似8Ba=2gtz+?wx的就是初始密码

进入mysql

shell> mysql -uroot -p

修改密码

mysql> ALTER USER 'root'@'localhost' IDENTIFIED BY 'MyNewPass4!';

密码总长至少8字符,一大写一小写一个数字一个特殊字符


其他补充

mysql账号授权及参考 官网授权传送门

CREATE USER 'jeffrey'@'localhost' IDENTIFIED BY 'mypass';
GRANT ALL ON db1.* TO 'jeffrey'@'localhost';
GRANT SELECT ON db2.invoice TO 'jeffrey'@'localhost';
ALTER USER 'jeffrey'@'localhost' WITH MAX_QUERIES_PER_HOUR 90;

远程登录

mysql> use mysql;
mysql> update user set host = '%' where user = 'jeffrey';

这一块的话5.7我直接创建远程账号似乎有问题不能用,密码机制应该做过调整,所以直接用官网方法创建了账号,并通过修改host为%来允许远程登录。

防火墙

nginx安装完毕后,mysql安装完毕后,如果开启防火墙的看官可能还会遇到不能访问,但是本地运行正常的问题,那么就有可能是防火墙的问题了。
由于我使用的是firewall防火墙,仅描述一下firewall添加规则方式。
添加方式

shell> firewall-cmd --add-service=http --permanent
shell> firewall-cmd --add-service=https --permanent
shell> firewall-cmd --add-service=mysql --permanent

关闭方式

shell> firewall-cmd --remove-service=http --permanent

显示success就是成功了,添加完毕需要重启防火墙。

shell> systemctl restart firewalld

好好的理了一下,下次重装就能顺利多了。
