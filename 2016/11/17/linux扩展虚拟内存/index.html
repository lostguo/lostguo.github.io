linux扩展虚拟内存

发表于 2016-11-17 | 分类于 linux | 阅读次数

因为机子内存小，以前安装symfony时就因为内存不够安不上，回顾下命令。

如何安装

查看当前使用情况

# free -m

结果如下

#                total        used        free      shared  buff/cache   available
# Mem:            992         412          73           0         507         438
# Swap:             0           0           0

查看系统

# df -h

结果如下

# /dev/vda1        40G  5.8G   32G  16% /
# devtmpfs        488M     0  488M   0% /dev
# tmpfs           497M     0  497M   0% /dev/shm
# tmpfs           497M  344K  497M   1% /run
# tmpfs           497M     0  497M   0% /sys/fs/cgroup
# tmpfs           100M     0  100M   0% /run/user/0

创建空间

# dd if=/dev/zero of=/swap_increased bs=1024 count=1024000

结果如下

# 1024000+0    records    in
# 1024000+0    records    out
# 1048576000    bytes (1.0 GB) copied, 3.14372 s, 334 MB/s

设置为虚拟

# mkswap /swap_increased

立刻启用

# swapon /swap_increased

检查如下

#               total        used        free      shared  buff/cache   available
# Mem:            992         413          71           0         507         437
# Swap:           999           0         999

设置永久生效

# echo "/swap_increased swap swap defaults 0 0" >> /etc/fstab

如何增加

dd增加

# dd if=/dev/zero of=/tmp/swap bs=1024 count=1024000

mkswap制作一个swap文件

# mkswap /tmp/swap

swapon启动swap文件

# swapon /tmp/swap

如何删除

停止分区

# swapoff /swap_increased

删除文件

# rm -rf /swap/swap

删除”/etc/swap”指定文件

最后

# sed -i "/'\/swap_increased swap swap defaults 0 0'//" /etc/fstab
