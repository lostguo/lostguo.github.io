我的centos7 L2TP VPN 之旅

0.0苹果的IOS和mac系统在升级之后都移除了pptp,所以在下以前搭建的pptp就派不上用场了。啊,伟大的墙,只能含泪重新搭建。


准备工作

服务器及系统

作者使用的是阿里云ECS香港服务器

shell> cat /etc/redhat-release
CentOS Linux release 7.2.1511 (Core)

更新yum源

shell> yum update

安装epel源

shell> yum install epel-release -y

依赖组件

shell> yum install -y openswan ppp pptpd xl2tpd wget


正式配置

配置ipsec

shell> vim /etc/ipsec.conf

修改为如下内容

# /etc/ipsec.conf - Libreswan IPsec configuration file

# This file:  /etc/ipsec.conf
#
# Enable when using this configuration file with openswan instead of libreswan
#version 2
#
# Manual:     ipsec.conf.5

# basic configuration
config setup
    # NAT-TRAVERSAL support, see README.NAT-Traversal
    nat_traversal=yes
    # exclude networks used on server side by adding %v4:!a.b.c.0/24
    virtual_private=%v4:10.0.0.0/8,%v4:192.168.0.0/16,%v4:172.16.0.0/12
    # OE is now off by default. Uncomment and change to on, to enable.
    oe=off
    # which IPsec stack to use. auto will try netkey, then klips then mast
    protostack=netkey
    force_keepalive=yes
    keep_alive=1800

conn L2TP-PSK-NAT
    rightsubnet=vhost:%priv
    also=L2TP-PSK-noNAT

conn L2TP-PSK-noNAT
    authby=secret
    pfs=no
    auto=add
    keyingtries=3
    rekey=no
    ikelifetime=8h
    keylife=1h
    type=transport
    left=你的外网IP 例 47.90.*.**
    leftid=你的外网IP
    leftprotoport=17/1701
    right=%any
    rightprotoport=17/%any
    dpddelay=40
    dpdtimeout=130
    dpdaction=clear
    ike=3des-sha1,aes-sha1,aes256-sha1,aes256-sha2_256
    phase2alg=3des-sha1,aes-sha1,aes256-sha1,aes256-sha2_256
    sha2-truncbug=yes
# For example connections, see your distribution's documentation directory,
# or the documentation which could be located at
#  /usr/share/docs/libreswan-3.*/ or look at https://www.libreswan.org/
#
# There is also a lot of information in the manual page, "man ipsec.conf"

# You may put your configuration (.conf) file in the "/etc/ipsec.d/" directory
# by uncommenting this line
#include /etc/ipsec.d/*.conf

leftid和left的内容填写上你的外网IP地址

配置公钥PSK

shell> vim /etc/ipsec.secrets

改为如下

#include /etc/ipsec.d/*.secrets
yourIP %any: PSK "your pre-shared-key"

其中yourIP是你的外网ip地址

配置pptp

shell> vim /etc/pptpd.conf

改为如下

#ppp /usr/sbin/pppd
option /etc/ppp/options.pptpd
#debug
# stimeout 10
#noipparam
logwtmp
#vrf test
#bcrelay eth1
#delegate
#connections 100
localip 10.0.1.2
remoteip 10.0.1.50-245

localip为vpn所持有的虚拟网段的ip，这里设置localip为2，把1留给了xl2tpd。
remoteip为vpn所提供出去的虚拟网段的ip。

配置xl2tp

shell> vim /etc/xl2tpd/xl2tpd.conf

改为如下内容

;
; This is a minimal sample xl2tpd configuration file for use
; with L2TP over IPsec.
;
; The idea is to provide an L2TP daemon to which remote Windows L2TP/IPsec
; clients connect. In this example, the internal (protected) network
; is 192.168.1.0/24.  A special IP range within this network is reserved
; for the remote clients: 192.168.1.128/25
; (i.e. 192.168.1.128 ... 192.168.1.254)
;
; The listen-addr parameter can be used if you want to bind the L2TP daemon
; to a specific IP address instead of to all interfaces. For instance,
; you could bind it to the interface of the internal LAN (e.g. 192.168.1.98
; in the example below). Yet another IP address (local ip, e.g. 192.168.1.99)
; will be used by xl2tpd as its address on pppX interfaces.
[global]
; ipsec saref = yes
listen-addr = 47.90.*.**
auth file = /etc/ppp/chap-secrets
port = 1701
[lns default]
ip range = 10.0.1.50-10.0.1.245
local ip = 10.0.1.1
refuse chap = yes
refuse pap = yes
require authentication = yes
name = L2TPVPN
ppp debug = yes
pppoptfile = /etc/ppp/options.xl2tpd
length bit = yes

其中listen-addr填写的是你的外网ip;
ip range为所指定的VPN对外提供的虚拟IP的范围;
local ip为VPN所持有的为l2tp使用的虚拟IP。

配置pptp的参数

shell> /etc/ppp/options.pptpd

内容改为如下

# Authentication
name pptpd
#chapms-strip-domain

# Encryption
# BSD licensed ppp-2.4.2 upstream with MPPE only, kernel module ppp_mppe.o
# {{{
    refuse-pap
    refuse-chap
    refuse-mschap
    # Require the peer to authenticate itself using MS-CHAPv2 [Microsoft
    # Challenge Handshake Authentication Protocol, Version 2] authentication.
    require-mschap-v2
    # Require MPPE 128-bit encryption
    # (note that MPPE requires the use of MSCHAP-V2 during authentication)
    require-mppe-128
    # }}}

# OpenSSL licensed ppp-2.4.1 fork with MPPE only, kernel module mppe.o
# {{{
    #-chap
    #-chapms
    # Require the peer to authenticate itself using MS-CHAPv2 [Microsoft
    # Challenge Handshake Authentication Protocol, Version 2] authentication.
    #+chapms-v2
    # Require MPPE encryption
    # (note that MPPE requires the use of MSCHAP-V2 during authentication)
    #mppe-40    # enable either 40-bit or 128-bit, not both
    #mppe-128
    #mppe-stateless
    # }}}

ms-dns 8.8.4.4
ms-dns 8.8.8.8

#ms-wins 10.0.0.3
#ms-wins 10.0.0.4

proxyarp
#10.8.0.100

# Logging
#debug
#dump
lock
nobsdcomp
novj
novjccomp
nologfd

配置l2tp的参数

shell> vim /etc/ppp/options.xl2tpd

内容如下

#require-pap
#require-chap
#require-mschap
ipcp-accept-local
ipcp-accept-remote
require-mschap-v2
ms-dns 8.8.8.8
ms-dns 8.8.4.4
asyncmap 0
auth
crtscts
lock
hide-password
modem
debug
name l2tpd
proxyarp
lcp-echo-interval 30
lcp-echo-failure 4
mtu 1400
noccp
connect-delay 5000
# To allow authentication against a Windows domain EXAMPLE, and require the
# user to be in a group "VPN Users". Requires the samba-winbind package
# require-mschap-v2
# plugin winbind.so
# ntlm_auth-helper '/usr/bin/ntlm_auth --helper-protocol=ntlm-server-1 --require-membership-of="EXAMPLE\VPN Users"'
# You need to join the domain on the server, for example using samba:
# http://rootmanager.com/ubuntu-ipsec-l2tp-windows-domain-auth/setting-up-openswan-xl2tpd-with-native-windows-clients-lucid.html

配置用户名和密码

shell> vim /etc/ppp/chap-secrets

内容如下

# Secrets for authentication using CHAP
# client        server  secret                  IP addresses
gu******45      pptpd   da*******               *
gu******45      l2tpd   da*******               *
ch******45      l2tpd   da*******               *

中间不是空格请使用tab

设置配置,允许IP转发

vim /etc/sysctl.conf

内容如下

net.ipv4.ip_forward = 1
net.ipv4.conf.all.rp_filter = 0
net.ipv4.conf.default.rp_filter = 0
net.ipv4.conf.eth0.rp_filter = 0
net.ipv4.conf.all.send_redirects = 0
net.ipv4.conf.default.send_redirects = 0
net.ipv4.conf.all.accept_redirects = 0
net.ipv4.conf.default.accept_redirects = 0

重新加载配置

shell> sysctl -p

配置防火墙

shell> vim /usr/lib/firewalld/services/pptpd.xml

内容如下

<?xml version="1.0" encoding="utf-8"?>
<service>
  <short>pptpd</short>
  <description>I want fly the wall</description>
  <port protocol="tcp" port="1723"/>
</service>

修改l2tpd.xml

shell> vim /usr/lib/firewalld/services/l2tpd.xml

内容如下

<?xml version="1.0" encoding="utf-8"?>
<service>
  <short>l2tpd</short>
  <description>L2TP IPSec</description>
  <port protocol="udp" port="500"/>
  <port protocol="udp" port="4500"/>
  <port protocol="udp" port="1701"/>
</service>

最后将服务加入防火墙

firewall-cmd --reload
firewall-cmd --permanent --add-service=pptpd
firewall-cmd --permanent --add-service=l2tpd
firewall-cmd --permanent --add-service=ipsec
firewall-cmd --permanent --add-masquerade
firewall-cmd --permanent --direct --add-rule ipv4 filter FORWARD 0 -p tcp -i ppp+ -j TCPMSS --syn --set-mss 1356
firewall-cmd --reload

命令 xl2tpd -D 可以看到控制台的输出信息。
命令 vim /var/log/secure 可以查看相关日志。
命令 systemctl restart/start/status/stop pptpd ipsec xl2tpd firewalld 可以开启重启关闭服务以及查看服务状态。

添加开机启动

shell> systemctl enable pptpd ipsec xl2tpd


结语

一般来说到此就可以科学上网了。作者使用的是mac+iphone皆可正常使用。希望各位也能顺利的安装使用。
mac如果不能连接的话,有可能是排序的问题。
网络偏好设置->设置->设定服务顺序->进行拖动排序
