我的前端之路以及入坑vue

不知道为什么好像走向了全栈路线，作为一个全栈工程师，我怎么可以不会使用前端框架呢。谨以此文记录一下vue的使用过程。

历史回顾

相对来说，我的主要注意力还是集中在了后端，所以对于前端，并没有进行太多系统性的总结，也并没有涉及太多的项目，都是服务于后台。

客串js交互

在我从事第一份工作的时候，对于前后端职责的划分稀里糊涂，前端sisy同志会把一些表单提交和一些循环的之类的互动js留给我，尽管我是个phper，前端之路算是从这开始了。
写过不少风格奇诡的js，主要涵盖的是业务逻辑ajax交互之类的，有时候自己也想写写前端代码，但是丑爆了，记得那时响应式火起来没多久的一年，但是那会儿有更多的php代（没）码（有）需（研）要（究）研（出）究（啥），所以没去研究。

客串前端

后来第二份工作也就是目前就职的公司，是一家创业型公司。运营总监每天找我要数据，可麻烦了，果断嘿咻嘿咻搭建了公司管理后台，找了点模板再自己写写样式，虽然丑爆了但是顶用了，css+js都自己写，此时仍然未使用bootstrap之类的东西。

技术衍变之bootstrap

再之后公司要做官网，但是没前端，怎么办，我顶上。反正官网简单，4个页面，写写样式啥的，都不用后台，用2天写完了，但看着有点不舒服，手机看起来也会有各种问题，当时强迫症就上来了，去找响应式办法，还是想到了bootstrap，直接操刀子上。嘿咻嘿咻一天写完了，速度挺快，问题还少。从此之后一发不可收拾，重构公司管理后台，虽然还是丑丑的，但是起码是有正常三观的网站了，让自己舒心了不少。后面陆续使用这套方式写了后台的采购模块，微信商城等。

技术衍变之angularjs

直到某一天感觉bootstrap好像写手机感觉一般，没有weui适合微信，然后是js的问题了，之前都用的php+twig/smarty 或者think自带的模板，可能因为和安卓前端配合多了，变得不咋喜欢服务器渲染这一套，说是前后端分离，但是不够彻底。
于是又开始琢磨起来，正值webapp概念较火，又值追书神器看书收费，网站各种小广告，便用angularjs+bootstrap做了个简单的看书软件（含自动采集指定书籍，目录，与阅读等基础功能）。

技术衍变之vue

vue是在接触了angularjs后了解到的，作为一个轻型的前端框架，文档也挺齐全，又值公司需要搭建erp系统，在此不断纠结到底深入angular还是深入vue，选了vue，但当遇见问题的时候总是想着要不换回angular，但是最后客服了犹豫，用劲的啃了下去。又结合node+npm+webpack体系，最终还是做了一套东西出来，基础的坑基本踩好了。比如router需要在main.js进行注册这种问题也会坑上不少时间。

一如既往的附上安装教程菜鸟教程、官网教程

创建项目

进入到自己的工程文件目录

# vue init webpack-simple 工程名字<工程名字不能用中文>

或者创建 vue1.0 的项目

# vue init webpack-simple#1.0 工程名字<工程名字不能用中文>

一路回车，Author写上你的大名

cd进入你的工程目录

安装依赖

# npm install

安装vue-router(路由系统，控制访问地址的)、vue-resource(用来数据请求)

# cnpm install vue-router vue-resource --save

进入调试模式(一般会让你自动浏览诸如localhost:8081/**)之类的

# npm run dev

安装饿了么的前端组件库

# cnpm install element-ui@next -S

这几个是用来控制样式的，不然引入element后报错

# cnpm install style-loader --save-dev
# cnpm install css-loader --save-dev
# cnpm install file-loader --save-dev

几个报错

  1. [Vue warn]: Error when rendering component <ElRow>

    Vue 2.1.5 将 _h 重命名为 _c，而 Element 目前发的版本都是用以前的 compiler 编译的，导致新版 runtime无法运行Element。目前的解决方案是锁定 Vue 的版本为 2.1.4

  2. 对于服务器端来说，不了解jsonp的同志建议看一下jsonp的概念，因为不知道所以坑了，vue-resource请求这边会自带一个callback参数，而服务器端也是需要返回这个传去的参数的，不然会报错，错到你怀疑自己用错语法。

  3. 通用方法，我想要用一个通用组件的方法怎么办

    Vue.prototype.yourFunctionName(){}

    然后可以在子组件中this.yourFunctionName()进行调用。

几个小知识

  1. js中不要用=> 可以替代为function(){}

  2. 通用方法搞死我了，子组件post请求，在不能及时获取情况的时候

  3. sessionStorage.someYourkey='xxx'存储在客户端这边的类似session的信息。

  4. v-mode="aaa"，绑定变量aaa

  5. :class="***"，设置class

  6. 因为想用element-ui新版的表单展开那个功能，然后把vue升级到了2.1.10，享受npm带来的方便，其实报错的自己的update一下一般不会有太大问题。 - -下次需要注意新版本的发布，搞了半天功能不能用结果是版本问题。醉醉的。

  7. vue外链方法，直接使用import的方式import入js，这个方法比较好

    import aa from “./component/jquery” 之类的
    import bb from “./component/mqtt”

    局部使用的话可以封个组件

    <remote-js src="https://code.jquery.com/jquery-3.1.1.js"></remote-js>

    components: {
  'remote-js': {
    render(createElement) {
      return createElement('script', { attrs: { type: 'text/javascript', src: this.src }});
    },
    props: {
      src: { type: String, required: true },
    },
  },
}

乱入php开启GIZP压缩开启方式

if( !headers_sent() && // 如果页面头部信息还没有输出
    extension_loaded("zlib") && // 而且php已经加载了zlib扩展
    strstr($_SERVER["HTTP_ACCEPT_ENCODING"],"gzip")) //而且浏览器接受GZIP
{
    ini_set('zlib.output_compression', 'On');
    ini_set('zlib.output_compression_level', '4');
}
