Laravel 第一节 环境搭建及路由使用

发表于 2017-06-29 | 分类于 laravel | 阅读次数:

趁着这段时间，总结一下laravel。

5.4 laravel的环境要求

PHP >= 5.6.4
OpenSSL PHP Extension
PDO PHP Extension
Mbstring PHP Extension
Tokenizer PHP Extension
XML PHP Extension


安装方式

使用 Composer 下载 Laravel 安装包

composer global require "laravel/installer"

需要你将 ~/.composer/vendor/bin 路径加到 PATH，只有这样系统才能找到 laravel 的执行文件。

一旦安装完成，就可以使用 laravel new 命令在指定目录创建一个新的 Laravel 项目，例如：laravel new blog 将会在当前目录下创建一个叫 blog 的目录，此目录里面存放着新安装的 Laravel 和代码依赖。这个方法的安装速度比通过 Composer 安装要快上许多：

laravel new blog

因为代码依赖是直接一起打包安装的。

通过 Composer Create-Project

composer create-project --prefer-dist laravel/laravel blog


本地访问

php artisan serve

这个字体好像好看

font-family: 'Raleway', sans-serif;


控制器

生成控制器

php artisan make:controller Interface/InterfaceController --resource


目录说明

我们的业务代码基本都放在了app目录下


采用正式配置

修改nginx的root为laravel项目的public文件夹，并设置storage目录为777


路由玩法

routes文件夹里设置路由，一般在routes文件夹web.php中设置。

1.请求方式

请求类型有如下：

  • Route::get($uri, $callback);
  • Route::post($uri, $callback);
  • Route::put($uri, $callback);
  • Route::patch($uri, $callback);
  • Route::delete($uri, $callback);
  • Route::options($uri, $callback);

2.普通get请求

Route::get('user', function () {
    return "user-laravel";
});

3.多种请求允许

Route::match(['get', 'post'], 'user', function () {
    return "user-laravel";
});

//允许所有请求方式

Route::any('user', function () {
    return "user-laravel";
});

4.参数请求

Route::get('user/{id}', function ($id) {
    return 'user_id:'.$id;
});

//多参数

Route::get('user/{id}/comments/{comment}', function ($id, $commentID) {
    return 'user_id:'.$id.',comment_id:'.$commentID;
});

//可选参数
Route::get('user/{name?}', function ($name = null) {
    return $name;
});

5.正则约束

Route::get('user/{name}', function ($name) {

    return $name;

})->where('name', '[A-Za-z]+');

//全局约束

找不到匹配项会报错。

6.命名路由
命名路由后，直接调用route('profile')，后续修改路由也不会影响调用，利于维护。

Route::get('user/profile', function () {
    return route('profile');
})->name('profile');

//重定向，方法里执行
return redirect()->route('profile');

针对待参数的命名路由，需要进行如下调用：

Route::get('user/{id}/profile', function ($id) {

})->name('profile');

//如下使用 
$url = route('profile', ['id' => 1]);

7.路由组

中间件路由 待具体使用
要给路由组中定义的所有路由分配中间件，可以在路由组中使用 middleware 键，中间件将会依照列表内指定的顺序运行

Route::group(['middleware' => 'auth'], function () {
    Route::get('/', function ()    {
        // 使用 `Auth` 中间件
    });

    Route::get('user/profile', function () {
        // 使用 `Auth` 中间件
    });
});

路由前缀 prefix 键可以给每个路由组中的路由加上指定的 URI 前缀

Route::group(['prefix' => 'demo'], function () {
    Route::get('/', function ()    {
        return 'It is auth\'s root ';
    });

    Route::get('user/profile', function () {
        return 'It is auth\'s user/profile ';
    });
});

子域名路由 待具体使用

Route::group(['domain' => '{account}.myapp.com'], function () {
    Route::get('user/{id}', function ($account, $id) {
        //
    });
});

命名空间 待具体使用

Route::group(['namespace' => 'Admin'], function () {
    // 在 "App\Http\Controllers\Admin" 命名空间下的控制器
});
这时使用 namespace 参数来指定组内所有控制器的公共命名空间

8.获取当前路由信息

$route = Route::current();

$name = Route::currentRouteName();

$action = Route::currentRouteAction();


其他命令

php artisan route:cache

运行这个命令之后，每一次请求的时候都将会加载缓存的路由文件。
记住，如果添加了新的路由，你需要刷新路由缓存。
因此，你应该只在项目部署时才运行 route:cache 命令

你可以使用 route:clear 命令清除路由缓存：

php artisan route:clear
