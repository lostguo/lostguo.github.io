hexo更换next，并同时部署到coding

发表于 2017-07-13 | 分类于 blog | 阅读次数

晤，还是觉得next好看，比较喜欢简约派。

next主题官方教程

下载next主题

进入站点文件夹

cd your-hexo-site

从github上clone，会比较新

git clone https://github.com/iissnan/hexo-theme-next themes/next

主题切换

修改站点_config.yml文件

theme: next

题中题

next有三种风格供你选择

Muse    - 默认 Scheme，这是 NexT 最初的版本，黑白主调，大量留白
Mist    - Muse 的紧凑版本，整洁有序的单栏外观
Pisces  - 双栏 Scheme，小家碧玉似的清新

比如我喜欢Mist风格

cd themes/next

选择

#scheme: Muse
scheme: Mist
#scheme: Pisces

此时刷新可以看到博客风格已经变了。

头像设置

在_config.yml中进行如下设设置

avatar: http://www.example.com/**.jpg

添加分类

不添加的话会请求不到晤

hexo new page categories

在source/categories目录的index.md中修改:

title: categories
date: 2017-07-13 17:31:56
type: "categories"
---

添加标签

hexo new page categories

在source/tags目录的index.md中修改:

title: tags
date: 2017-07-13 17:32:11
type: "tags"
---

社交标签

修改主题_config.yml

social:
  GitHub: https://github.com/lostguo

social_icons:
  enable: true
  GitHub: github
  Twitter: twitter
  Weibo: weibo

友情链接

# Blogrolls
links_title: 友情链接
#links_layout: block
#links_layout: inline
links:
  Johnny Luv: http://blog.starpoetry.cn/

图片文章

---
title: 博客切为next主题
date: 2017-07-13 17:32:42
tags:  [hexo,blog]
category: blog
photos:
 - http://bruce.u.qiniudn.com/2013/11/27/reading/photos-0.jpg

其他方式

新建hexo/scaffolds/photo.md:

---
layout: {{ layout }}
title: {{ title }}
date: {{ date }}
tags: 
photos: 
---

然后可以通过命令

hexo new photo "photoPostName"

来生成照片文章。

文章阅读量

注册账号

去leancloud.cn注册一个账号

新建应用

新建一个Counter的应用，并选择无限制权限

取得APP ID、APP KEY

选择应用后，设置/应用key中可以查看

填写入主题_config.yml中

leancloud_visitors:
  enable: true
  app_id: 91cRYBUXXdjP6Sc5GTuTvqus-gzGzoHsz
  app_key: o2e2e4Ga0BGHAoSeWxr173L5

访问量的唯一标识符是文章的发布日期以及文章的标题，更改了会清零，请注意。

安全性

为保证安全性，在设置的安全中心设置安全域名。

Q&A?

不晓得为什么，部署好了之后一直提示401错误

Failed to load resource: the server responded with a status of 401 () 
us-api.leancloud.cn/1.1/classes/Counter

可以去看看这个Issues

我的next版本还是最新的，然后移除掉themes/next/layout/_third-party/analytics/lean-analytics.swig中

<script>AV.initialize("{{theme.leancloud_visitors.app_id}}", "{{theme.leancloud_visitors.app_key}}");AV.useAVCloudUS();</script>

AV.useAVCloudUS();这一串去掉竟然成功了，我也不是很清楚这是为什么。

coding设置

注册coding账号

去coding.net注册一个账号，此过程不作赘述。

添加id_rsa.pub

可以参考https://coding.net/help/doc/git/ssh-key.html

mac打开

open ~/.ssh

将此公钥添加到coding上

测试连接

ssh -T git@git.coding.net

如果出现类似

Coding.net Tips : [Hello lostguo, You've connected to Coding.net via SSH successfully!]

那么就成功了

hexo提交配置

在_config.yml中进行如下配置

deploy:
  type: git
  repository:
      github: https://github.com/lostguo/lostguo.github.io.git,master
      coding: https://git.coding.net/lostguo/lostguo.git

然后就可以hexo g、hexo d咯

coding的pages服务部署

直接在coding网站的代码/Pages服务设置部署来源即可

然后成功的运行在了https://lostguo.coding.me/lostguo额

去除跳转

首先升级成银牌会员，补充资料即可升级。

然后再网站底部添加

<p>Hosted by <a href="https://pages.coding.me" style="font-weight: bold">Coding Pages</a></p>

抑或图片版，按需选择即可
